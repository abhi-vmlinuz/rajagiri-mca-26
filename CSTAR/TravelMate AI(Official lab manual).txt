TravelMate AI
Building Your First AI Agent using Tool Calling

Introduction
In this workshop, we will build a simple AI Travel Assistant using Python and
the Groq API. Unlike a normal chatbot that only generates text, our AI agent
can use external tools (functions) to answer user queries with real data.
The Travel Assistant can:


Check the weather for a city.



Search available flights.



Recommend hotels.



Suggest popular tourist attractions.

During this workshop, we will learn how to:


Create custom Python functions (tools).



Register those tools with an AI model.



Allow the AI to decide which tool to use based on the user's question.



Execute the selected tool and return the result to the AI.



Generate a final, user-friendly response.

By the end of this session, you will have built a basic agentic AI application that
demonstrates how Large Language Models can interact with external tools to
solve real-world tasks.

5. Creating AI Tools
What is a Tool?
A tool is simply a Python function that the AI model is allowed to use.
Instead of generating every answer itself, the AI can call these functions
whenever additional information is needed.
In this project, we will create four tools:


Get Weather



Search Flights



Find Hotels



Get Attractions

Tool 1 – Get Weather
Add the following function after all the dictionary data .
def get_weather(city):

weather = weather_data.get(city)

if weather is None:
return "Sorry, weather information is not available."

return (
f"City: {city}\n"
f"Temperature: {weather['temperature']}\n"
f"Condition: {weather['condition']}\n"
f"Humidity: {weather['humidity']}"
)

Understanding the Code
Line 1
def get_weather(city):
This creates a Python function named get_weather.
It accepts one parameter called city.
Whenever someone asks for weather information, this function will be
executed.

Line 2
weather = weather_data.get(city)
The .get() method searches the dictionary using the city name.
For example,
weather_data.get("Goa")
returns
{
"temperature":"31°C",
"condition":"Sunny",
"humidity":"70%"
}
If the city does not exist,
weather_data.get("Paris")
returns
None
instead of causing an error.

Line 4
return "Sorry, weather information is not available."

Returns a friendly message to the user.

Remaining Lines
The remaining code creates a formatted string using Python f-strings.
Instead of returning the raw dictionary, it produces readable output such as:
City: Goa
Temperature: 31°C
Condition: Sunny
Humidity: 70%

6. Connecting to the Groq API
Now that we have created our travel database and Python tools, we need to
connect our application to a Large Language Model (LLM).
In this workshop, we will use Llama 3.3 70B, which is hosted by Groq.
The model will analyze the user's question and decide which tool should be
executed.

Step 1 – Load the API Key
Add:
load_dotenv()
Explanation:
load_dotenv() loads the API key stored in the .env file so Python can access it.

Step 2 – Create the AI Client
Add:
client = OpenAI(
api_key=os.getenv("GROQS_API_KEY"),
base_url="https://api.groq.com/openai/v1"
)

Explanation:



OpenAI() creates the API client.



api_key reads your API key from the .env file.



base_url tells the SDK to use Groq's API instead of OpenAI's.

8. Sending the User's Question to the AI
The following code allows the user to enter a travel-related question and sends
it to the LLM.
The AI analyzes the question and decides whether one of the registered tools
should be used.

Next copy paste this:

user_question = input("You: ")

#Ask the ai a question

response = client.chat.completions.create(
model="llama-3.3-70b-versatile",
messages=[
{
"role": "user",
"content": user_question
}
],
tools=tools,
tool_choice="auto"
)
message = response.choices[0].message

# Did the AI choose a tool?
if message.tool_calls:

tool_call = message.tool_calls[0]

tool_name = tool_call.function.name
arguments = json.loads(tool_call.function.arguments)

# Execute the correct tool
if tool_name == "get_weather":

tool_result = get_weather(arguments["city"])

elif tool_name == "search_flights":
tool_result = search_flights(arguments["destination"])

elif tool_name == "find_hotels":
tool_result = find_hotels(arguments["city"])

elif tool_name == "get_attractions":
tool_result = get_attractions(arguments["city"])

else:
tool_result = "Unknown tool."

print("\n===== TOOL RESULT =====")
print(tool_result)
print("=======================\n")
messages = [
{
"role": "user",
"content": user_question
},
message,
{
"role": "tool",
"tool_call_id": tool_call.id,

"content": tool_result
}
]

final_response = client.chat.completions.create(
model="llama-3.3-70b-versatile",
messages=messages
)

print("TravelMate AI:\n")
print(final_response.choices[0].message.content)

Explanation


input() accepts the user's question.



client.chat.completions.create() sends the question to the Groq LLM.



tools=tools gives the model access to all the registered tools.



tool_choice="auto" lets the model automatically decide which tool to
use.

If the model selects a tool:


The tool name and arguments are extracted.



The corresponding Python function is executed.



The tool's result is stored in tool_result.
